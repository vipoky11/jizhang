# 默认初始数据说明

## 📋 概述

**注意**：系统默认情况下**不插入任何初始数据**，所有分类和账户都需要用户手动添加。

首次启动时，系统只会创建空的数据库表结构，用户需要：
1. 通过"分类管理"添加分类
2. 通过"账户管理"添加账户
3. 然后才能开始记账

---

## 🏷️ 分类数据

系统**不会自动创建任何分类**，用户需要手动添加。

### 建议的分类示例

如果需要参考，可以添加以下分类：

**支出分类：**
- 餐饮、交通、购物、娱乐、医疗、教育、其他支出

**收入分类：**
- 工资、奖金、投资收益、其他收入

---

## 💳 账户数据

系统**不会自动创建任何账户**，用户需要手动添加。

### 建议的账户示例

如果需要参考，可以添加以下账户：
- 微信、支付宝、现金、银联、其他

**注意**：可以设置其中一个账户为默认账户，新建交易记录时会自动选择。

---

## 📝 数据初始化逻辑

### 初始化行为

系统首次启动时：
1. ✅ **创建数据库表结构**（transactions, categories, accounts, suppliers, memos, tags）
2. ❌ **不插入任何数据**（分类、账户等都需要手动添加）

### 相关文件

- SQLite 初始化：`server/database/init-sqlite.js`
- MySQL 分类表：`server/database/add_categories_table.sql`
- MySQL 账户表：`server/database/add_accounts_table.sql`

---

## 🔄 初始化时机

### 自动初始化

1. **SQLite（桌面客户端）**
   - 首次启动 Electron 应用时自动创建表结构
   - 不会插入任何数据

2. **MySQL（开发环境）**
   - 运行 `npm run migrate-categories` 时创建分类表（不插入数据）
   - 运行 `npm run migrate-accounts` 时创建账户表（不插入数据）

### 手动添加数据

所有数据都需要通过应用界面手动添加：
1. 打开"分类管理"，添加分类
2. 打开"账户管理"，添加账户
3. 然后才能开始记账

---

## ✏️ 添加数据

### 通过界面管理（推荐）

启动应用后，在界面中：
- **分类管理**：添加/编辑/删除分类
- **账户管理**：添加/编辑/删除账户

### 如果需要恢复默认数据

如果需要恢复之前的默认数据逻辑，可以：
1. 修改 `server/database/init-sqlite.js`
2. 修改 `server/database/add_categories_table.sql`
3. 修改 `server/database/add_accounts_table.sql`
4. 添加 INSERT 语句

---

- **初始账户数量**：0 个（需要手动添加）

## ⚠️ 注意事项

1. **首次使用**：需要先添加至少一个分类和一个账户，才能开始记账
2. **必须添加分类**：记账时分类为必填项
3. **建议添加账户**：虽然账户有默认值，但建议至少添加一个账户
4. **默认账户**：可以设置一个账户为默认账户，新建交易记录时会自动选择
5. **数据管理**：所有数据都可以通过界面添加、编辑、删除

---

## 🔍 查看当前数据

可以通过以下方式查看当前数据：

1. **界面查看**：在应用中打开"分类管理"和"账户管理"
2. **数据库查询**：
   ```sql
   SELECT * FROM categories ORDER BY sort_order;
   SELECT * FROM accounts ORDER BY sort_order;
   ```
